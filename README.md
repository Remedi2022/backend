# REMEDi

## 1. 서류 발급 필요 없는 보험금 청구 자동화, REMEDi

REMEDi는 서류 발급과 보험금 청구 과정을 생략하여 환자에게 편리함을 제공하고자 하는 프로젝트입니다.
<br/>
[HL7 EHC_E01](https://hl7-definition.caristix.com/v2/HL7v2.7/TriggerEvents/EHC_E01) 국제 표준 메세지를 적용하여 보험금 청구를 자동화하는 프로세스를 구축해봤습니다.

<details>
<summary>REMEDi 이름 알아보기</summary>
<div markdown="1">
: REMEDi는 원격을 의미하는 remote와 의료를 의미하는 medical에서 따온 이름입니다.
<br/>처리 방안, 해결책, 치료약이라는 뜻을 가진 단어 remedy와도 유사하여,
<br/>불편한 보험금 청구 과정을 해결해 줄 서비스라는 의미 또한 담았습니다.
</div>
</details>

<details>
<summary>REMEDi 아이디어 배경</summary>
<div markdown="1">

### 보험 청구 간편화 및 자동화

![스크린샷 2022-12-17 오전 12 49 15](https://user-images.githubusercontent.com/89639470/208136362-99d9940c-dc64-4525-b14e-2854ab36d163.png)

: 병원에서만 구입 가능한 MD(Medical Device) 제품에 실손의료보험이 적용되는 사실을 모르고, 보험금 받지 못하는 환자들이 많습니다.
<br/>추가로, 환급 받기 위해 병원에 직접 방문하여 필요 제출 서류를 발급받아야 하는 번거로움이 있습니다.
<br/>1만원 이하의 진료비는, 보험금 미청구 건수가 51.4%입니다.
<br/>이러한 문제들을 해결하기 위해 과정들을 자동화하는 시스템을 가정, 설계, 구현해보는 프로젝트를 기획했습니다.

</div>
</details>

## 3. System design & Backend Process Flow

<div markdown="1">

<details>
<summary>[병원 EMR 기능] Creating HL7 Message & Publish</summary>
<div markdown="1">
: EMR의 기본 흐름은, 환자 등록 → 접수 → 진료 → 수납의 과정을 거치며, 클라우드 기반 EMR에 의무적으로 기록이 저장됩니다.
<br/>수납이 완료됨과 동시에, EMR의 데이터베이스에서 실비보험금 청구에 필요한 정보들을 추출,
<br/>이를 HL7 국제 표준에 따라 Message를 생성하고, 보험사 서버로 전송(Kafka-Publish)합니다.
</div>
</details>

<details>
<summary>[보험사 서버 기능] HL7 Message Subscribe & Restore</summary>
<div markdown="1">
: 보험사 서버는, Apache NiFi로 대체하여 기획했습니다.
<br/>보험금 청구에 필요한 정보들이 HL7 Message 형태로 수신(Kafka-Subscribe)되면,
<br/>보험사 서버에서는 Message를 Parsing 후, 각각의 정보를 데이터베이스에 저장합니다.
<br/>보험 심사를 위해, 이를 보험사 웹 페이지에서 정보를 확인할 수 있게 구현했습니다.
</div>
</details>
